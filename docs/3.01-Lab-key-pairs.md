
# Lab: Creating Key Pairs in AWS Management Console

This guide walks you through creating an EC2 Key Pair using the AWS Management Console. Key Pairs are the standard mechanism for securely accessing EC2 instances.

---

## What is a Key Pair?

A **Key Pair** in AWS consists of a public key (stored by AWS) and a private key (downloaded to your machine). It's used to authenticate securely to EC2 instances:

   - **Linux**: Used for SSH authentication.
   - **Windows**: Used to decrypt the Administrator password for RDP (Remote Desktop Protocol) access.

---

## Prerequisites

- An AWS account with permissions to manage EC2 resources.
- A web browser.

---

## Procedure

### 1. Sign in to AWS Management Console

1. Open [https://aws.amazon.com/](https://aws.amazon.com/) in your browser.
2. Click **Sign In to the Console**.
3. Enter your credentials and complete MFA if prompted.

---

### 2. Navigate to EC2 Key Pairs

1. From the AWS Console home page, type **EC2** in the search bar and select **EC2**.
2. In the left sidebar, find the **Network & Security** section.
3. Click **Key Pairs**.

---

### 3. Create a New Key Pair

1. Click the **Create key pair** button in the top-right corner.
2. Fill in the following fields:
   - **Name**: Choose a unique, descriptive name (e.g., `MyLabKeyPair`).
   - **Key pair type**:
     - **RSA**: Supported for both Linux and Windows instances. AWS uses 2048-bit SSH-2 RSA keys.
     - **ED25519**: Offers strong security and better performance, but is **only supported for Linux instances**. Do not use ED25519 if you plan to connect to Windows instances.
   - **Private key file format**:
     - **.pem**: Works with OpenSSH clients on Linux, macOS, and modern Windows (10/11 with built-in OpenSSH).
     - **.ppk**: Required for PuTTY on Windows.
3. *(Optional)* Add **tags** to help organize and track your resources.
4. Click **Create key pair**.

### 4. Download and Store the Private Key

- Your private key file (`.pem` or `.ppk`) will download automatically once the key pair is created.
- Store the file somewhere secure. Never share it or commit it to source control.

---

**Note: AWS does not retain your private key. If the file is lost, it cannot be recovered; you'll need to create a new key pair.**

---

### 5. Verify Key Pair Creation

- Your new key pair will appear in the Key Pairs list.
- You can attach this key pair to EC2 instances at launch time.
- For SSH access on Linux/Unix, restrict the file permissions before use:
  ```sh
  chmod 400 /path/to/MyLabKeyPair.pem
  ```
> View Permissions: `ls -l <File_Name>`
- For SSH access on Windows 10/11. restrict the file permission before use:
   - Reset and remove all inherited permissions
>Change directory `cd ~\Downloads`
```powershell
icacls.exe "C:\path\to\MyLabKeyPair.pem" /inheritance:r
```
   - Grant 'Read' access to only the current user
```powershell
icacls.exe "C:\path\to\MyLabKeyPair.pem" /grant:r "$($env:USERNAME):R"
```
>View Permissions:`Get-Acl <Key_Name>`

- For PuTTY on Windows, use the `.ppk` file directly, or convert a `.pem` file using PuTTYgen.

---

## Recommendations

- **Security**: Keep your private key strictly private. Restrict file level access to your user only.
- **Backup**: Store a copy in a secure, encrypted location such as a password manager or encrypted USB drive.
- **Naming**: Use names that reflect the project, environment, and owner (e.g., `prod-web-david`).
- **Tagging**: Tag key pairs consistently to simplify resource management and cost allocation.
- **Windows instances**: Always use RSA; ED25519 is not supported for Windows.

---

## Cleanup

Remove key pairs you no longer need to reduce your attack surface:

1. Select the key pair from the list.
2. Click **Actions** > **Delete**.

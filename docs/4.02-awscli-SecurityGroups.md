# AWS CLI-EC2 Security Group Operations

> Note: When running commands in PowerShell, remember to use `aws.exe` instead of `aws`.

---
- First, configure your AWS credentials in Terminal or PowerShell by running:
```
aws configure
```
---

**Get first VPC_ID (Linux/Unix)**

```bash
VPC_ID=$(aws ec2 describe-vpcs --query "Vpcs[0].VpcId" --output text)
```

**Show VPC_ID**

```bash
echo "$VPC_ID"
```

**Create security group (Linux/Unix)**

- Syntax:
```bash
SG_JSON=$(aws ec2 create-security-group --group-name <SECURITY_GROUP_NAME> --description "<DESCRIPTION>" --vpc-id $VPC_ID)
```
- Example:
```bash
SG_JSON=$(aws ec2 create-security-group --group-name My_Web_Security_group  --description "Allowing SSH connections to the developers" --vpc-id $VPC_ID) 
```

<img width="1470" height="221" alt="Screenshot 2026-05-03 at 11 31 30 AM" src="https://github.com/user-attachments/assets/9632a38c-b0d7-4731-a9ba-66993acfb569" />


**Extract security group ID**

```bash
SG_ID=$(echo "$SG_JSON" | jq -r '.GroupId')
```
```bash
echo $SG_ID
```

**Authorize SSH ingress**

```bash
aws ec2 authorize-security-group-ingress --group-id <SECURITY_GROUP_ID> --protocol tcp --port 22 --cidr 0.0.0.0/0
```
- Example:
```bash
aws ec2 authorize-security-group-ingress --group-id sg-0ac81deba064f08a2 --protocol tcp --port 22 --cidr 0.0.0.0/0 
```

**Authorize SMTP (IPv6)**

> `--cidr` only accepts IPv4. Use `--ipv6-cidr-block` for IPv6 addresses.

```bash
aws ec2 authorize-security-group-ingress \
  --group-id $SG_ID \
  --ip-permissions '[{"IpProtocol":"tcp","FromPort":25,"ToPort":25,"Ipv6Ranges":[{"CidrIpv6":"::/0"}]}]'
```

> When using `--protocol -1` (all protocols), do **not** include a `--port` argument; AWS CLI will return an error if you do.

**Authorize SMTP (IPv6)**

> `--cidr` only accepts IPv4. Use `--ipv6-cidr-block` for IPv6 addresses.

```bash
# Write JSON to a temp file
'[{"IpProtocol":"tcp","FromPort":25,"ToPort":25,"Ipv6Ranges":[{"CidrIpv6":"::/0"}]}]' | Out-File -FilePath ip-perms.json -Encoding ascii

# Use file:// reference
aws ec2 authorize-security-group-ingress --group-id $SG_ID --ip-permissions file://ip-perms.json
```

## Delete security group

Syntax:
```bash
aws ec2 delete-security-group --group-id <SECURITY_GROUP_ID>
```

Example:
```bash
aws ec2 delete-security-group --group-id $SG_ID
```
